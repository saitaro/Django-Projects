Парсер логов веб-сервера ngnix

Часть 1:

  • создать скрипт с названием nginx_log_parser.py
  • в скрипте мы должны только один раз делать итерацию по строкам файла и желательно всего одну итерацию на всю программу
  • записать в отдельный файл все запросы с кодом 500
  • вывести print количество запросов с кодом 200 и методом GET (к примеру 25)
  • вывести  print список уникальных ip адресов (через запятую, к примеру 127.0.0.1, 192.168.0.1, 192.222.32.11)

Часть 2:

  1. создаем пользователя в СУБД с паролем
  2. создаем базу данных с этим пользователем как владельцем базы
  3. создаем таблицу в базе данных c столбцами КОД (число), IP(строка), МЕТОД(строка)
  4. в коде парсера подключаемся к базе
  5. при парсинге каждой строки пишем в базу код http-ответа в КОД, ip в IP, метод запроса в МЕТОД. (названия заменяем на английские)

Часть 3:

  1. Делаем очистку базы каждый раз перед записью лога.
  2. передаем один аргумент при вызове команды из shell http://www.pythonforbeginners.com/argv/more-fun-with-sys-argv
  • если fill - наполняем базу (то что делали в части 2)
  • если count - выводим кол-во сколько уникальных IP, METHOD и CODE у нас в базе. (IP: 22, METHOD: 4, CODE:5)
